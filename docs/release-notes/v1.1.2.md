# Fan App 1.1.2

Fan App 1.1.2 fixes the live power figure on Macs that are fully charged and connected to AC power.

## What changed

- Power usage now follows macOS live system-load telemetry instead of relying only on battery current.
- The value continues to refresh every 30 seconds while the Fan App popover is open.
- Voltage and battery current remain available as a fallback on Macs that do not expose system-load telemetry.
- Added regression tests for both AC-powered and battery-powered readings.

## Why it was showing 0.0 W

A fully charged battery reports zero charging or discharging current. The previous calculation multiplied that battery current by voltage, which produced 0.0 W even while the Mac was drawing power from its charger. Fan App now reads the Mac's current system load directly.

## Verified behavior

The corrected reading was verified on a fully charged Mac connected to AC power. It changed from 24.6 W to 27.1 W across consecutive monitoring cycles, confirming that the displayed value updates with live load.

## Install

1. Download `fan-1.1.2-macos.dmg`.
2. Open the disk image and drag `fan.app` into Applications.
3. Right-click `fan.app` and choose **Open** on first launch.
4. Choose **Install helper** and approve the narrowly scoped fan-control helper when prompted.

The supplied build targets Apple Silicon and macOS 26.1 or later. It is ad-hoc signed, so macOS may display the standard warning for independently distributed apps.

SHA-256 checksum files are attached beside both downloads.
